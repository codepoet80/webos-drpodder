drPodder Redux is a podcast download and listener app for webOS devices. It is derrived work of Guttenpodder and drPodder, a well known podcast app by Jamie Hatfield.

Usage

  • Add your feeds
  • Download episodes you're interested in
  • Play Podcasts!

Feed list

  • the feed list is the first screen you see after starting drPodder (not counting the loading screen)
  • Feel free to delete any of the default podcasts added during first startup. Swipe them away as usual in webos.
  • You can add podcasts by url, search for podcasts or scan for existing media files on your device by using the '+' icon in the bottom left on the feed screen
    • To enter a feed url directly, choose "Enter Feed URL"
    • To find a Podcast feed by title use "Search Directory"
    • "Dynamic Playlist" creates a virtual feed. That's a selectable mashup of feeds already existing in drPodder
    • "Add local media" searches your device for existing media files (new in drPodder 0.9.0). You can enter expressions to filter by media title and/or file path. The comparison is case insensitive. Thanks to Adam Marks for his article on webOS Nation about Guttenpodder's local media feature (written as of GP 0.9.1).
      Sadly, local media function work only on webos 2.x and higher. At first use, you may be asked for permisson to access the file system of your device (this is called "media permission"). Some webos 2.x devices have got a problem here; they do not ask, but do not allow access either.
      In this case you may run an application to fix this problem; you'll find it at http://dl.dropbox.com/u/32016000/com.palm.app.permissions-reset_1.0.0_all.ipk
  • Refresh button to refresh the list of feeds
  • Feed artwork should show up on first update, if the feed has one
  • Line under feed name contains the first episode name in the feed (depending on the sort order of that feed)
  • 3 mini icons
    • disk - number of episodes downloaded/saved
    • star - number of new episodes (their status is set to Unlistened)
    • pause - number of new episodes you've started listening to (count currently not updating, but bookmarks are retained)
  • Tap the mini icon area for feed operations: (Mark Listened & Edit Feed)
  • Swipe to delete
  • Reorder is supported

Episode List Screen

  • Grayed out episodes are listened to
  • Button on right:
    • Blank - listened to, and no file is available
    • Green Down Arrow - Unlistened (new) episode, ready for download
    • Red X - Cancel current download
    • Dark Blue Play - Episode ready to be played
    • Red Minus - Episode has been listened to, file still exists
  • Tap the button on the right for relevent menu choices
    • Download, play (all), Resume, Restart, Mark [Un]Listened, Episode Details, Delete
    • Restart is the only method for going into an episode and starting from the beginning. All other methods will resume playback from your last saved position
  • The Episodes are ordered by publiction dates, with the newest at top. You can change the sorting in the feed preference (new in drPodder 0.9.0)
  • You can reorder the episodes manually, by tap and dragging them (the usual webos way) (new in drPodder 0.9.0, fixed in 0.9.3).

Episode Details Screen

  • Episode title
  • Playback scrubber with current time and time remaining and has a buffer indicator also
  • Episode Description (scrollable)
  • Play/Pause, +/-20 seconds, +/- 1 minute button
  • All playback operations *should* save your current position
  • command menu contains settings for sleep timer (new in 0.9.3)

Some Feature

Auto update / refresh

In the preference screen you can select to auto update at certain intervalls.

  • Set the auto update switch to "on",
  • select one feed or playlist in the "update feed" drop down list (or select "[all]" for all feeds)
  • select "daily" in the update type drop down list
  • select a time (e.g. 4:45 am)
  • depending on the prices of your mobile provider you should set "DL only over WiFi" to "on", so sound file download will only happens if wifi is connected.
  • it's a good idea to set "Autoload" to "on" for those feeds you hear allways (see feed setting)
and every morning you pick up your webOS device, you've got already your fresh podcasts.

    What is a podcast?

    A podcast is a type of digital media consisting of an episodic series of audio, video, PDF, or ePub files subscribed to and downloaded through web syndication or streamed online to a computer or mobile device (Source: Wikipedia article "Podcast")

    What is a feed?

    A feed is a short file containig description, the url and some other computer readable information about podcast episodes.

    How do I delete a feed?

    As everywhere in webOS: Swipe it away (tap and hold the feed, and - without lifting the finger - swipe it to the right). You'll be asked for confirmation.

    How Do I migrate from drPodder or podFrenzy to drPodder?

    The easy way: use the Import/Export-Feature with file drpodder.xml or the clipboard. You will loose the state of the feeds, but not the feeds themself.
    The other way involves coping some internal database. It will keep you all feeds, state and downloads. Hopefully. See this posting and this article for an idea how to do it.

    How do I play all Episodes in a row?

    In the episode list tap once on the title bar (that thing with the name of the feed). The menu hided there has options to play all episodes from top or from bottom.

    Or you tap an episode a little longer, and select "play all from here"

    I start downloading my episodes, they get about halfway done, then stop downloading and give me an error.

    Is there any chance your media partition is full? (check under Device Info-Phone-Available)

    Why did you steal drPodder?

    I didn't. Jamie Hatfield put his app under the "GNU public licence" (GPL). So anybody is allowed to copy and change his app as long as this copied app is put under the same or compatible licence. And the source of the new app must be published (see this page). The big advantage: If an application is discontinued by the original author, the user is not left with a starving system (at least, if a new maintainer is found).
    See http://www.gnu.org/licenses/gpl-3.0.txt for the licence. And this post may help, too.

    Which feature are planned?

    A lot of features are impossible, because webOS does not support it. E.g. Speedup of podcast episodes or warning on filessystem low space. There are some ideas, but I do not promise anything (error fixing has priority):
    • display duration in episode list
    • skip epsiode button in episode detail
    • using the cam/photo picker for albumart
    • enhance "edit feed" scene to select the playlists the feed is visible in
    • enhance feed option to start episodes at fix offsets (to skip long or boring teasers)
    • enhance feed option to filter episodes for url, description or duration
    • unlikely: syncing with gpodder.net
    • fix translations :)
